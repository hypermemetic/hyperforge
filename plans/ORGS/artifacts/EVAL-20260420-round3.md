## What happened between round 2 and round 3

Round 2 declared PROMOTE. That was wrong. The round 1 revisions had
over-pinned the tickets — struct bodies, file paths, attribute syntax,
and exact message substrings were all treated as contract when they
were implementation. The evaluator (me) missed this because the
`epic-evaluation` skill did not yet carry the
capability-vs-implementation check.

The user flagged it directly:

> "don't the tickets talk about pinning concepts and capabilities and
> not implementations? maybe that's not clear enough. what I mean is
> the ticket building skill, and the epic one should discuss the fact
> that code in a ticket is only reserved when it's the easiest way to
> represent the ask."

The correct response was to **fix the methodology, then re-revise the
tickets at the capability level**, not to patch the tickets in
isolation.

## Ticket changes this round

Both ORGS-2 and ORGS-3 were rewritten so the `## Required behavior`
section reads as a spec of *what the system does* rather than a
prescription of *how the code is laid out*.

What was removed:
- The pinned `OrgSshKeys` struct body (name, fields, derives, serde
  attributes, file location).
- Method signatures as Rust code.
- `HyperforgeEvent::Info` / `HyperforgeEvent::Error` variant names.
- Exact message substring contracts (`"exists"`, `"merged"`, etc.).
- `type` tag values (`"info"` / `"error"`).

What replaced them:
- Capability statements about the param shape (*"a typed record with
  one optional field per known forge"*) and the consequences (*Synapse
  emits per-forge flags; unknown forges are rejected at parse time*).
- Outcome statements about what a caller observes (*"failure event
  identifies the failure as 'already exists'"*) without pinning the
  discriminator.
- Round-trip integrity as a first-class capability.

Acceptance criteria were rewritten to check observable state
(filesystem persistence round-tripped via `orgs_list`, event-stream
observability, parse-layer rejection) rather than source-level
artifacts.
