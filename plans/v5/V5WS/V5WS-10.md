## Problem

V5WS comprises nine implementation tickets (V5WS-2..9). Epic readiness
is observable only when all their scripts pass AND each V5WS-1 user
story maps to at least one green script. This ticket codifies the map
and aggregates the scripts. It introduces no new behavior.

## Required behavior

The checkpoint script runs every `tests/v5/V5WS/V5WS-{2..9}.sh` and
reports a state-of-epic map per user story. Each story is classified
green / yellow / red with a one-liner:

- **green**: every script the story depends on passes.
- **yellow**: all dependent scripts run, but at least one skipped under the current tier set (e.g. V5WS-9 under default tier-1-only).
- **red**: at least one dependent script fails, or one is missing.

Story-to-script map (asserted by the checkpoint):

| Story | Scripts |
|---|---|
| U1 (stand up a workspace) | V5WS-4.sh, V5WS-2.sh, V5WS-3.sh |
| U2 (rename, reconcile) | V5WS-8.sh |
| U3 (delete dir, reconcile) | V5WS-8.sh |
| U4 (remove with cascade) | V5WS-7.sh, V5WS-5.sh |
| U5 (sync workspace) | V5WS-9.sh |
| U6 (cross-org workspace) | V5WS-6.sh, V5WS-3.sh, V5WS-7.sh |

## What must NOT change

- No new implementation. The checkpoint script only invokes sibling scripts; it adds no new assertions outside the story-to-script aggregation.
- The story-to-script map above is the contract. Changing it requires a ticket revision.
- `tests/v5/V5WS/V5WS-9.sh` is tier 2; under tier-1-only runs, U5 is yellow, not red.

## Acceptance criteria

1. `bash tests/v5/V5WS/V5WS-10.sh` under tier-1-only execution: returns non-zero if any of V5WS-2..8 fails; skips V5WS-9 and classifies U5 as yellow with a one-liner naming the tier-2 gate.
2. Under a tier-2-enabled run (invoked with the env var `HF_TIER2=1`), V5WS-10 executes V5WS-9.sh and counts it toward U5. U5 is green iff V5WS-9.sh exits 0.
3. The output contains exactly one line per user story in the form `U<n> <green|yellow|red>: <one-liner>`.
4. The checkpoint script performs no RPC calls itself — every observation is a sibling script exit code.
5. Missing sibling script (not executable) is red, not a bash crash; the missing filename appears in the one-liner.

## Completion

- Run `bash tests/v5/V5WS/V5WS-10.sh` → exit 0 under tier-1 (U5 yellow acceptable) and under tier-2 (U5 green required).
- Status flips when every blocked_by ticket is complete AND this script exits 0 under the current tier policy.
